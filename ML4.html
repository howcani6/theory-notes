Foundations

1. 확률, 파라미터, 데이터의 의미

왜 이 개념을 먼저 보는가

MLE, MAP, Naive Bayes, Logistic Regression, Linear Regression은 전부 결국 데이터를 보고 파라미터를 추정하는 문제로 연결된다. 따라서 먼저 \(\theta\)와 \(D\)가 무엇인지, 그리고 likelihood와 posterior가 무엇인지 분명하게 잡아야 한다.

\(\theta\)와 \(D\)의 뜻

파라미터 \(\theta\)

모델 안에 숨어 있는 수치적 규칙이다. 직접 관측되지 않지만 데이터 생성 구조를 결정한다.

데이터 \(D\)

우리가 실제로 관측한 결과이다. 보통 학습에서는 데이터만 보이고, 파라미터는 직접 보이지 않는다.

어떤 진짜 파라미터 \(\theta\)가 있고, 그 \(\theta\)에 의해 데이터 \(D\)가 생성되었다고 생각하는 것이 전체 출발점이다.

likelihood와 posterior의 차이

$$P(D\mid\theta)$$

이 식은 “\(\theta\)라고 가정했을 때 데이터 \(D\)가 얼마나 그럴듯한가?”를 묻는다. 즉, 원인을 먼저 고정하고 결과를 본다.

$$P(\theta\mid D)$$

이 식은 “데이터 \(D\)를 본 뒤에 어떤 \(\theta\)를 더 믿게 되는가?”를 묻는다. 즉, 결과를 보고 원인에 대한 믿음을 업데이트한다.

많이 헷갈리지만, \(P(D\mid\theta)\)와 \(P(\theta\mid D)\)는 일반적으로 같지 않다. 방향이 다르기 때문이다.

베이즈 정리로 연결

$$P(\theta\mid D)=\frac{P(D\mid\theta)P(\theta)}{P(D)}$$

posterior는 likelihood와 prior를 결합해 만들어진다. 따라서 posterior를 이해하려면 likelihood뿐 아니라 prior도 함께 이해해야 한다.

헷갈리기 쉬운 포인트

  • likelihood는 \(\theta\)에 대한 함수로 볼 때의 이름이지, 항상 확률분포처럼 해석하는 것은 아니다.
  • 파라미터 \(\theta\)를 확률변수로 볼 수 있는지는 관점에 따라 다르다. 이 차이가 곧 Frequentist와 Bayesian의 차이로 이어진다.
다음 연결: 이제 “그러면 실제로 \(\theta\)를 어떻게 고를 것인가?”라는 질문으로 넘어가며, 그 대표 답이 MLE와 MAP이다.
Estimation

2. MLE와 MAP

왜 이 개념을 보는가

데이터를 보고 파라미터를 추정하는 방법에는 여러 가지가 있지만, 그중 가장 대표적인 것이 MLE와 MAP이다. 이 둘을 이해하면 뒤에서 나오는 regularization까지 자연스럽게 연결된다.

MLE

$$\hat{\theta}_{\mathrm{MLE}}=\arg\max_\theta P(D\mid\theta)$$

의미는 분명하다. “내가 관측한 데이터를 가장 잘 설명하는 \(\theta\)를 고르자.” 즉, 데이터만 보고 가장 그럴듯한 파라미터를 찾는다.

MAP

$$\hat{\theta}_{\mathrm{MAP}}=\arg\max_\theta P(\theta\mid D)$$

베이즈 정리를 쓰면

$$\hat{\theta}_{\mathrm{MAP}}=\arg\max_\theta P(D\mid\theta)P(\theta)$$

가 된다. 즉 MAP는 데이터의 설명력과 사전 믿음을 동시에 반영한다.

prior의 의미

$$P(\theta)$$

prior는 데이터를 보기 전부터 가지고 있던 \(\theta\)에 대한 믿음이다. 예를 들어 동전이라면 \(\theta=0\)이나 \(\theta=1\) 같은 극단적 값보다 \(\theta=0.5\) 근처를 더 그럴듯하다고 보는 믿음을 prior로 넣을 수 있다.

비교 항목 MLE MAP
목적함수 \(P(D\mid\theta)\) \(P(D\mid\theta)P(\theta)\)
데이터 외 정보 사용 안 함 prior 사용
적은 데이터 극단적 추정 가능 더 안정적일 수 있음
뒤 연결 likelihood 중심 regularization과 연결

핵심 해석

  • MLE: 데이터만 보고 고른다.
  • MAP: 데이터도 보되, 파라미터가 원래 어떨 것 같은지에 대한 믿음도 함께 반영한다.

헷갈리기 쉬운 포인트

  • MAP는 posterior 전체를 쓰는 true Bayesian prediction과 다르다. posterior의 최고점 하나만 뽑는 것이다.
  • prior가 들어간다고 해서 무조건 더 좋은 것은 아니다. prior가 잘못되면 결과를 왜곡할 수 있다.
다음 연결: 왜 MLE는 likelihood를 보고, MAP는 posterior를 보는지가 철학적으로 어디서 나오는지 이해하려면 Frequentist와 Bayesian의 차이를 봐야 한다.
Philosophy

3. Frequentist vs Bayesian

왜 이 구분이 필요한가

MLE와 MAP는 단순히 계산법 차이가 아니라, 파라미터를 무엇으로 보느냐에 대한 철학 차이에서 나온다.

Frequentist

  • \(\theta\)는 고정된 진짜 값
  • 우리는 그 값을 모를 뿐
  • 그래서 \(P(D\mid\theta)\)를 중심으로 본다

Bayesian

  • \(\theta\)도 불확실성의 대상
  • 따라서 \(P(\theta)\), \(P(\theta\mid D)\)가 의미 있다
  • 데이터를 보면서 믿음을 업데이트한다

수식 관점에서의 차이

Frequentist에게는 \(\theta\)가 확률변수가 아니므로 \(P(\theta)\)를 직접 다루지 않는 것이 자연스럽다. 반면 Bayesian에게는 \(\theta\)도 확률모형 안에 들어가므로 posterior가 자연스럽다.

Frequentist는 “정답은 이미 하나로 정해져 있다”고 보고, Bayesian은 “정답에 대한 믿음이 분포로 존재한다”고 본다.

어디로 이어지는가

  • MLE는 Frequentist 감각과 강하게 연결된다.
  • MAP와 posterior prediction은 Bayesian 감각과 연결된다.
  • 뒤에서 regularization을 prior로 해석할 수 있게 된다.
다음 연결: 이제 이 차이를 가장 단순하게 보여주는 예제가 Coin Tossing이다. 여기서 likelihood, prior, posterior, MLE, MAP가 한 번에 연결된다.
Canonical Example

4. Coin Tossing은 왜 핵심 예제인가

왜 이 예제를 보는가

Coin Tossing은 매우 단순하지만, 우리가 뒤에서 계속 쓰는 거의 모든 개념을 압축해서 보여준다. 즉 likelihood, MLE, prior, posterior, MAP, pseudo-count, true Bayesian prediction까지 다 들어 있다.

모델

앞면 확률을 \(\theta\)라고 두면

$$P(H)=\theta,\qquad P(T)=1-\theta$$

앞면이 \(n_H\)번, 뒷면이 \(n_T\)번 나왔다면 likelihood는

$$P(D\mid\theta)=\binom{n_H+n_T}{n_H}\theta^{n_H}(1-\theta)^{n_T}$$

이다.

수식의 의미

이 식은 앞면이 많이 나올수록 \(\theta\)가 큰 쪽이 유리하고, 뒷면이 많이 나올수록 \(1-\theta\)가 큰 쪽이 유리하다는 뜻이다. 즉 관측한 빈도가 가능한 파라미터를 조여 가는 구조다.

MLE의 의미

로그를 취해 정리하면 결국 MLE는 관측 비율이 된다.

$$\hat\theta_{\mathrm{MLE}}=\frac{n_H}{n_H+n_T}$$

즉 데이터만 보면 “앞면이 나온 비율”이 가장 그럴듯한 파라미터가 된다.

MAP와 Beta prior

prior를

$$\theta\sim\mathrm{Beta}(\alpha,\beta)$$

로 두면

$$P(\theta)\propto \theta^{\alpha-1}(1-\theta)^{\beta-1}$$

이고, posterior는 다시 Beta family 안에 남는다. 이 구조를 통해 prior가 pseudo-count처럼 작동한다는 해석이 가능해진다.

pseudo-count 해석

Beta prior는 마치 미리

  • \(\alpha-1\)개의 가짜 head
  • \(\beta-1\)개의 가짜 tail

을 본 것처럼 작동한다. 그래서 적은 데이터에서 MLE가 너무 극단적으로 가는 것을 완화해준다.

true Bayesian prediction과 MAP의 차이

$$P(H\mid D)=\int P(H\mid\theta)P(\theta\mid D)\,d\theta$$

true Bayesian prediction은 posterior 전체를 평균낸다. 반면 MAP는 posterior에서 가장 높은 점 하나만 택한다. 이 차이는 “점 추정”과 “분포 전체 사용”의 차이이다.

Coin Tossing에서는 계산이 쉬워서 개념이 잘 보인다. 그래서 뒤의 Naive Bayes, Logistic, Linear Regression까지도 이 예제의 감각을 계속 가져가면 이해가 훨씬 편하다.
다음 연결: 이제 파라미터 추정을 넘어, 입력 \(X\)가 주어졌을 때 정답 클래스 \(Y\)를 맞히는 분류 문제로 확장한다.
From Estimation to Classification

5. 분류 문제로 어떻게 넘어가는가

질문의 변화

앞에서는 “어떤 \(\theta\)가 데이터를 잘 설명하는가?”를 물었다. 이제는 질문이 조금 바뀐다.

입력 \(X\)가 주어졌을 때, 클래스 \(Y\)를 알고 싶다.
$$P(Y\mid X)$$

즉 분류의 목표는 posterior class probability를 알고 싶은 것으로 표현된다.

Bayes rule의 역할

$$P(Y\mid X)=\frac{P(X\mid Y)P(Y)}{P(X)}$$

직접 \(P(Y\mid X)\)를 모델링하기 어렵다면, 클래스별 데이터 생성 방식 \(P(X\mid Y)\)와 class prior \(P(Y)\)를 모델링한 뒤 Bayes rule로 뒤집을 수 있다.

  • \(P(Y)\): 각 클래스가 얼마나 자주 나타나는가
  • \(P(X\mid Y)\): 클래스가 주어졌을 때 데이터가 어떻게 생기는가
  • \(P(Y\mid X)\): 입력을 봤을 때 어느 클래스일 가능성이 큰가
다음 연결: 바로 이 generative 접근의 대표적인 모델이 Naive Bayes이다.
Generative Model

6. Naive Bayes

왜 보는가

Naive Bayes는 앞에서 본 확률 개념들이 실제 분류 문제에서 어떻게 사용되는지를 보여주는 대표적인 generative classifier다.

기본 아이디어

  • 먼저 \(P(Y)\)를 배운다.
  • 그다음 \(P(X\mid Y)\)를 배운다.
  • Bayes rule로 \(P(Y\mid X)\)를 계산한다.

즉 클래스가 데이터를 어떻게 생성하는지를 모델링한 뒤 분류한다.

naive assumption

$$P(X\mid Y)=\prod_{j=1}^d P(X_j\mid Y)$$

의미는 “클래스 \(Y\)가 주어지면 feature들은 서로 독립이다”이다.

왜 이 가정이 중요한가

원래 \(P(X\mid Y)\)는 고차원 joint distribution이라 다루기 어렵다. 그런데 독립 가정을 두면 여러 1차원 분포의 곱으로 분해되므로 계산이 매우 쉬워진다.

Naive Bayes의 핵심은 현실을 완벽하게 반영하는 것이 아니라, 강한 가정으로 tractable한 계산을 얻는 데 있다.

의사결정 규칙

$$\hat y=\arg\max_y P(y\mid x)=\arg\max_y P(x\mid y)P(y)$$

여기서 \(P(x)\)는 모든 클래스에 대해 공통이므로 비교에서는 빠진다.

종류

Categorical NB

feature가 범주형일 때 사용한다.

\(P(X_j=a\mid Y=c)\)

Multinomial NB

문서의 단어 count처럼 빈도형 feature에 사용한다.

\[P(x\mid c)\propto \prod_{j=1}^{d}\theta_{jc}^{x_j}\]
  • \(x_j\): \(j\)번째 단어의 등장 횟수
  • \(\theta_{jc}=P(\text{word }j\mid c)\): 클래스 \(c\)에서 단어 \(j\)가 나올 확률

Gaussian NB

연속형 feature를 Gaussian으로 모델링한다.

\(\mathcal N(\mu_{jc},\sigma_{jc}^2)\)

Multinomial NB의 수식 의미

$$P(x\mid m, y=c)=\frac{m!}{x_1!\cdots x_d!}\prod_{j=1}^d \theta_{jc}^{x_j}$$

여기서 조합항

$$\frac{m!}{x_1!\cdots x_d!}$$

은 같은 count vector를 만드는 여러 단어 순서의 개수를 세는 항이다. 즉 multinomial은 단순한 곱셈이 아니라 “그 count를 만들 수 있는 모든 순서”를 함께 고려한다.

Gaussian NB의 의미

$$P(x_j\mid y=c)=\mathcal N(\mu_{jc},\sigma_{jc}^2)$$
$$P(x\mid y=c)=\prod_{j=1}^d \mathcal N(x_j;\mu_{jc},\sigma_{jc}^2)$$

이는 클래스가 주어졌을 때 각 feature가 독립적인 Gaussian이라는 뜻이다.

공분산 대각행렬의 뜻

$$\Sigma_c=\mathrm{diag}(\sigma_{1c}^2,\dots,\sigma_{dc}^2)$$

off-diagonal 항이 0이라는 것은 feature 간 상관관계를 무시한다는 뜻이다. 즉 Gaussian NB는 클래스 내부에서 feature들 사이의 공동 움직임을 보지 않는다.

장점과 한계

장점 한계
계산이 빠르고 간단함 독립 가정이 너무 강할 수 있음
적은 데이터에서도 잘 작동 가능 상관관계를 무시함
텍스트 같은 sparse 데이터에 강함 확률값 자체는 calibration이 안 좋을 수 있음
다음 연결: Naive Bayes는 generative 모델이지만, 특정 경우에는 결정경계가 선형처럼 보인다. 이 점이 Logistic Regression으로 넘어가는 중요한 다리다.
Bridge

7. Naive Bayes도 왜 선형적으로 볼 수 있는가

왜 이 포인트가 중요한가

Naive Bayes는 generative 모델이지만, 로그를 취해 클래스 점수를 비교하면 선형분류기처럼 보일 수 있다. 이 사실이 Logistic Regression으로 자연스럽게 이어진다.

$$\log P(y)+\sum_j x_j\log\theta_{jy}$$

두 클래스를 비교하면 결국

$$w^Tx+b$$

꼴로 정리될 수 있다. 즉 각 feature가 점수를 더하거나 빼는 방식으로 작동한다.

Naive Bayes가 “생성모델이면서도 선형 결정경계처럼 보일 수 있다”는 점이, 곧 “그렇다면 아예 \(P(Y\mid X)\)를 직접 선형적으로 모델링할 수 있지 않을까?”라는 질문으로 이어진다.

Gaussian NB의 경우에도 분산 구조가 단순하면 선형 경계가 나오고, 일반적으로는 quadratic 경계가 나올 수 있다. 즉 선형성은 모델 구조의 특별한 경우에서 나타난다.

다음 연결: 이제 generative 우회 없이, 아예 \(P(Y\mid X)\)를 직접 모델링하는 Logistic Regression으로 간다.
Discriminative Model

8. Logistic Regression

왜 보는가

Naive Bayes가 \(P(X\mid Y)\)를 모델링해서 \(P(Y\mid X)\)를 얻는 generative 모델이었다면, Logistic Regression은 처음부터 \(P(Y\mid X)\)를 직접 모델링하는 discriminative 모델이다.

기본 형태

$$P(y=1\mid x)=\sigma(w^Tx+b)$$
$$\sigma(z)=\frac{1}{1+e^{-z}}$$

선형 점수 \(w^Tx+b\)는 원래 \(( -\infty,\infty )\) 범위를 가지므로 확률이 될 수 없다. 그래서 sigmoid가 이를 \([0,1]\) 구간으로 눌러 준다.

왜 sigmoid를 쓰는가

  • 큰 양수 입력은 1에 가까운 확률로 간다.
  • 큰 음수 입력은 0에 가까운 확률로 간다.
  • 즉 선형 점수를 확률적 해석이 가능한 값으로 바꾼다.

\(y\in\{-1,+1\}\) 표기의 장점

$$P(y\mid x)=\frac{1}{1+e^{-y(w^Tx+b)}}$$

이렇게 쓰면 \(y(w^Tx+b)\) 하나로 해석이 깔끔해진다.

  • \(y(w^Tx+b)>0\): 맞게 예측
  • \(y(w^Tx+b)<0\): 틀리게 예측
  • 절댓값이 클수록 더 자신 있는 예측

이 값은 margin처럼 작동한다.

MLE 관점

데이터 \(D=\{(x_i,y_i)\}_{i=1}^n\)에 대해 MLE는

$$\hat w=\arg\max_w P(D\mid w)$$

이고, 음의 로그우도를 최소화하면 결국

$$\min_w \sum_{i=1}^n \log\big(1+\exp(-y_iw^Tx_i)\big)$$

형태가 된다. 이것이 logistic loss이다.

MAP 관점

예를 들어

$$w\sim\mathcal N(0,\sigma^2I)$$

라는 prior를 두면, MAP는

$$\min_w \sum_{i=1}^n \log(1+\exp(-y_iw^Tx_i))+\lambda\|w\|_2^2$$

가 된다. 즉 L2 regularized logistic regression과 같아진다.

  • Logistic의 MLE = logistic loss 최소화
  • Logistic의 MAP = logistic loss + L2 regularization

핵심 의미

regularization은 갑자기 인위적으로 붙는 게 아니라, Bayesian 관점에서는 prior가 자연스럽게 만들어내는 항으로 해석된다.

왜 곧바로 최적화로 가는가

Logistic loss는 지수와 로그가 섞인 비선형 함수이므로, 일반적으로 선형회귀처럼 닫힌형 해를 깔끔하게 구할 수 없다. 그래서 반복 최적화가 필요하고, 그 대표가 Gradient Descent이다.

다음 연결: 이제 “gradient 반대 방향으로 간다”는 말이 정확히 무엇인지, 그리고 왜 그게 의미가 있는지 본다.
Optimization

9. Gradient Descent

왜 필요한가

Logistic Regression처럼 closed form이 없는 모델에서는 반복적으로 loss를 줄여 가며 파라미터를 업데이트해야 한다. 그 기본 알고리즘이 Gradient Descent이다.

업데이트 식

$$w_{t+1}=w_t-\alpha\nabla \ell(w_t)$$
  • \(\nabla \ell(w_t)\): 가장 빨리 증가하는 방향
  • \(-\nabla \ell(w_t)\): 가장 빨리 감소하는 방향
  • \(\alpha\): learning rate 또는 step size

왜 gradient의 반대 방향인가

$$\ell(w+s)\approx \ell(w)+\nabla \ell(w)^Ts$$

Taylor 1차 근사에 따르면, 작은 이동 \(s\)를 했을 때 loss 변화는 대략 \(\nabla \ell(w)^Ts\)로 결정된다. 이 값을 가장 작게 하려면 \(s\)가 \(-\nabla \ell(w)\) 방향이어야 한다. 그래서 gradient 반대 방향이 국소적으로 가장 빠른 감소 방향이 된다.

stopping criteria

gradient norm

\(\|\nabla \ell(w_t)\|\approx 0\)

더 내려갈 방향이 거의 없다고 본다.

loss change

\(|\ell(w_{t+1})-\ell(w_t)|\approx 0\)

추가 업데이트로 개선이 거의 없다.

parameter change

\(\|w_{t+1}-w_t\|\approx 0\)

파라미터 업데이트가 사실상 멈췄다.

learning rate의 중요성

  • 너무 크면 optimum을 지나쳐 발산하거나 진동할 수 있다.
  • 너무 작으면 수렴이 매우 느려진다.

즉 Gradient Descent는 방향 정보뿐 아니라 step size 설정도 매우 중요하다.

다음 연결: 그런데 GD가 항상 잘 되는 것은 아니다. 왜 어떤 경우에는 수렴을 보장할 수 있고, 어떤 경우에는 어려운지를 보려면 convexity와 smoothness가 필요하다.
Theory

10. 수렴 이론: Convexity, Lipschitz, Taylor 기반 부등식

왜 이론을 보는가

Gradient Descent를 단순히 “잘 되는 방법”으로만 아는 것이 아니라, 어떤 조건 아래에서 왜 작동하는지 수학적으로 이해하기 위해 본다.

Convexity

$$f(\lambda x+(1-\lambda)y)\le \lambda f(x)+(1-\lambda)f(y),\qquad 0\le\lambda\le1$$

직관적으로는 그래프가 밥그릇처럼 휘어 있어 두 점을 잇는 선분이 그래프 위에 놓인다. 중요한 이유는 convex 함수에서는 local minimum이 global minimum이라는 점이다.

L-Lipschitz gradient

$$\|\nabla f(x)-\nabla f(y)\|\le L\|x-y\|$$

이것은 gradient가 너무 급격하게 변하지 않는다는 뜻이다. 즉 함수가 충분히 매끈하여, 현재 gradient 정보가 근처에서도 어느 정도 신뢰 가능하다는 의미다.

smoothness inequality

$$f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}\|y-x\|^2$$

이 식은 함수값이 접선 근사에서 크게 벗어나더라도, 그 오차가 이차항 정도로 제어된다는 뜻이다. 즉 “너무 험하지 않다”는 것을 정식화한 것이다.

왜 이차항이 나오는가

gradient 변화는 거리의 1차 비례로 제한되고, 그 차이를 위치 차이에 대해 적분하면 함수값 차이에는 제곱항이 등장한다. 그래서 gradient는 선형으로 변하고 함수값 오차는 이차로 누적된다는 감각이 중요하다.

수렴 증명의 두 claim

Claim 1: Convexity가 주는 lower bound

$$f(y)\ge f(x)+\nabla f(x)^T(y-x)$$

접선이 함수 아래에 있다는 뜻이다.

Claim 2: Smoothness가 주는 upper bound

$$f(y)\le f(x)+\nabla f(x)^T(y-x)+\frac{L}{2}\|y-x\|^2$$

접선 + 이차항이 함수 위를 잡는다는 뜻이다.

텔레스코핑의 역할

$$\|w_t-w^*\|^2-\|w_{t+1}-w^*\|^2$$

이런 형태를 여러 step에 대해 더하면 중간 항이 지워지고 처음과 끝만 남는다. 그래서 누적 감소량을 간단하게 정리할 수 있다.

대표 수렴률

$$f(w_k)-f(w^*)\le \frac{L\|w_0-w^*\|^2}{2k}$$

이는 convex + smooth 조건 아래 기본 Gradient Descent가 \(O(1/k)\) 속도로 최적값에 가까워진다는 뜻이다.

  • Convexity는 “엉뚱한 local minimum에 갇히지 않음”을 도와준다.
  • Smoothness는 “현재 gradient 정보를 믿고 한 걸음 갈 수 있음”을 도와준다.
  • 둘을 합치면 GD의 감소와 수렴을 정량적으로 말할 수 있다.
다음 연결: 기본 GD의 약점을 보완하려고 다양한 optimizer가 등장한다.
Practice

11. Optimizer들

왜 필요한가

기본 Gradient Descent는 단순하고 이론적으로 중요하지만, 실제로는 느리거나 진동할 수 있고, 좌표별 스케일 차이를 반영하지 못하는 경우가 많다.

Batch / SGD / Mini-batch

방법 특징 장단점
Batch GD 전체 데이터로 gradient 계산 정확하지만 비쌈
SGD 샘플 하나로 gradient 계산 빠르지만 noisy
Mini-batch SGD 작은 묶음으로 계산 실전에서 가장 흔한 절충안

Momentum

$$s_t=\beta s_{t-1}+\alpha g_t,\qquad w_{t+1}=w_t-s_t$$

이전 방향을 기억해 같은 방향으로는 가속하고, 왔다 갔다 하는 방향은 상쇄한다. 즉 관성을 넣어 진동을 줄이고 골짜기 방향으로 더 빨리 간다.

AdaGrad

좌표별로 누적된 gradient 크기를 반영해 step size를 조절한다. 많이 업데이트된 좌표는 더 조심스럽게, 드물게 업데이트된 좌표는 더 크게 움직이게 한다. sparse feature에 특히 유리하다.

RMSProp

AdaGrad는 누적합이 계속 커져 학습률이 너무 빨리 작아질 수 있다. RMSProp은 과거 gradient 제곱의 이동평균을 사용해 이 문제를 완화한다.

Adam

Momentum의 1차 모멘트와 RMSProp의 2차 모멘트를 결합한 방식이다. 실전에서 자주 쓰이는 기본 선택지이며, 일반적으로 안정적으로 잘 작동하는 편이다.

Newton Method

gradient뿐 아니라 Hessian까지 사용해 곡률 정보를 반영한다. 잘 맞는 상황에서는 매우 빠르지만, 2차 미분과 역행렬 계산 비용이 크다.

12. Linear Regression

왜 마지막에 선형회귀로 가는가

앞에서 분류 문제를 충분히 본 뒤, 연속값 예측 문제에서는 어떤 확률모형과 추정 방식이 나오는지 보기 위해 Linear Regression으로 간다. 여기서도 MLE와 MAP 관점이 그대로 살아 있다.

모델

$$y=w^Tx+b+\varepsilon$$
$$\varepsilon\sim\mathcal N(0,\sigma^2)$$

즉 출력의 평균은 선형이고, 오차는 Gaussian noise라고 가정한다.

확률적 의미

$$y_i\mid x_i\sim \mathcal N(w^Tx_i+b,\sigma^2)$$

선형회귀는 단순히 직선을 맞추는 기하학적 문제가 아니라, “평균은 선형이고 오차는 Gaussian이다”라는 확률모형이다.

MLE

$$P(D\mid w,b)=\prod_{i=1}^n \frac{1}{\sqrt{2\pi\sigma^2}}\exp\left(-\frac{(y_i-(w^Tx_i+b))^2}{2\sigma^2}\right)$$

로그를 취하면 결국

$$\min_w \sum_{i=1}^n (y_i-(w^Tx_i+b))^2$$

가 된다. 즉 Gaussian noise 가정 아래 MLE는 MSE / SSE / OLS 최소화와 같다.

왜 최대화가 최소화가 되는가

로그우도를 정리하면 앞에 마이너스가 붙은 제곱오차합이 나온다. 따라서 likelihood 최대화는 제곱오차합 최소화와 동치가 된다.

closed form

$$w^*=(X^TX)^{-1}X^Ty$$

조건이 맞으면 normal equation으로 해를 직접 구할 수 있다. 이것이 Logistic Regression과의 중요한 차이점이다.

MAP와 Ridge

prior를

$$w\sim\mathcal N(0,\tau^2I)$$

로 두면 MAP는

$$\min_w \sum_{i=1}^n (y_i-(w^Tx_i+b))^2+\lambda\|w\|^2$$

가 된다. 즉 MAP는 Ridge Regression으로 이어진다.

regularization의 해석

오차항은 데이터를 잘 맞추려는 힘이고, 규제항은 \(w\)가 너무 커지지 않게 하려는 힘이다. Bayesian 관점에서는 이 규제항이 곧 prior에서 자연스럽게 나온다.

$$\lambda\propto \frac{\sigma^2}{\tau^2}$$

즉 데이터 노이즈가 크거나, \(w\)는 원래 0 근처일 것이라는 믿음이 강할수록 regularization이 강해진다.

closed form이 있는데 왜 GD도 배우는가

  • 데이터가 크면 \(X^TX\) 계산이 비쌀 수 있다.
  • 역행렬 계산 비용과 메모리 부담이 크다.
  • 실전에서는 반복 최적화가 더 현실적일 수 있다.
전체 연결: 결국 Linear Regression도 확률모형, MLE, MAP, regularization, optimization이라는 같은 줄기 안에 있다.
Wrap-up

13. 최종 요약

  • 우리가 계속 붙잡고 있었던 중심 질문은 “데이터를 보고 숨겨진 파라미터를 어떻게 추정할 것인가?”였다.
  • \(P(D\mid\theta)\)는 likelihood이고, \(P(\theta\mid D)\)는 posterior이다.
  • MLE는 likelihood를 최대화하고, MAP는 likelihood와 prior를 함께 본다.
  • Frequentist는 파라미터를 고정된 값으로 보고, Bayesian은 파라미터에 대한 분포를 허용한다.
  • Coin Tossing은 likelihood, MLE, prior, posterior, MAP, true Bayesian prediction을 한 번에 보여주는 원형 예제다.
  • Naive Bayes는 \(P(X\mid Y)\)와 \(P(Y)\)를 모델링하는 generative classifier다.
  • Naive assumption은 feature 독립 가정이며, 계산 가능성을 준다.
  • Naive Bayes는 특정 조건에서 선형 결정경계처럼 보일 수 있고, 이것이 Logistic Regression으로 이어진다.
  • Logistic Regression은 \(P(Y\mid X)\)를 직접 모델링하는 discriminative model이다.
  • Logistic의 MLE는 logistic loss 최소화, MAP는 L2 regularization과 연결된다.
  • Logistic은 closed form이 없어서 Gradient Descent가 필요하다.
  • Gradient Descent의 핵심은 gradient 반대 방향이 국소적으로 가장 빠른 감소 방향이라는 점이다.
  • 수렴 이론에는 convexity, Lipschitz gradient, smoothness inequality, telescoping이 핵심 역할을 한다.
  • Optimizer들은 기본 GD의 느림, 진동, 고정 학습률 문제를 보완한다.
  • Linear Regression은 Gaussian noise 가정 아래 MLE가 MSE/OLS로, MAP가 Ridge로 이어진다.
  • 결국 regularization은 Bayesian prior의 해석을 가진다.
즉, 확률과 파라미터의 의미에서 시작해 MLE/MAP, Naive Bayes, Logistic Regression, Gradient Descent, Linear Regression까지 이어지는 전체 흐름은 전부 하나의 질문에서 출발한 것이다.
