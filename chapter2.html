Intro

이 장의 목적

이 장의 목표는 단 하나다.

“학습(learning)이란 것을 수학적으로 어떻게 정의할 것인가?”

아직 알고리즘 자체보다 환경, 데이터, 예측기, 오차를 먼저 정확하게 정의한다. 즉, 머신러닝을 설명하는 최소한의 수학적 프레임을 세우는 장이다.

Example

2.0 파파야 예제

“우리는 맛있는 파파야를 어떻게 고를 수 있는가?”

1) 문제의 본질

어떤 대상을 보고 라벨을 예측하는 것이 핵심이다. 진짜 정답은 실제로 먹어보기 전까지 알 수 없고, 우리는 과거에 본 예시를 이용해 새로운 대상을 맞혀야 한다. 이것이 지도학습의 기본 구조다.

2) 구성 요소

(A) 입력 공간과 특징

  • 대상: 파파야 하나하나를 \(x\)로 표기
  • 입력 공간: 모든 가능한 파파야들의 집합 \(\mathcal X\)
  • 특징: 색깔, 부드러움 같은 관찰 가능한 정보
\(x=(\text{color, softness})\)

(B) 출력 공간

  • 예측 결과는 맛있음/맛없음 같은 라벨
  • 이진 분류에서는 보통
$$\mathcal Y=\{0,1\}$$

(C) 정답 함수

세상 속에 숨겨진 진짜 규칙은

$$f:\mathcal X\to\mathcal Y$$

학습자는 이 규칙을 직접 볼 수 없다.

(D) 훈련 데이터

과거 관찰 기록은 특징과 라벨의 쌍으로 저장된다.

$$S=\{(x_1,y_1),\dots,(x_m,y_m)\}$$

3) 학습의 목표

훈련 데이터 \(S\)를 이용해 새로운 파파야가 왔을 때 잘 맞히는 예측 규칙 \(h\)를 만드는 것이다.

$$h:\mathcal X\to\mathcal Y$$

4) 개념 대조표

파파야 예제 학습 이론 개념 수학적 기호
파파야 하나인스턴스\(x\)
색상, 부드러움특징\(x\in\mathbb R^d\)
맛있음 / 맛없음라벨\(y\in\mathcal Y\)
진짜 맛의 규칙정답 함수\(f\)
관찰 기록들훈련 데이터\(S\)
최종 예측 규칙가설\(h\)
Framework

2.1 A Formal Model

직관을 수학적 프레임으로 바꾸는 단계다.

(1) Learner의 입력

① Domain set

$$\mathcal X$$
  • 라벨을 붙이고 싶은 대상들의 집합
  • 보통 각 \(x\in\mathcal X\)는 feature 벡터로 표현

② Label set

$$\mathcal Y=\{0,1\}\quad(\text{또는 }\{-1,+1\})$$
  • 이 장에서는 이진 분류만 다룬다

③ Training data

$$S=((x_1,y_1),\ldots,(x_m,y_m))\in(\mathcal X\times\mathcal Y)^m$$
  • 샘플 수는 \(m\)
  • 실제로는 집합이 아니라 순서 있는 시퀀스
  • 중복도 가능

(2) Learner의 출력

$$h:\mathcal X\to\mathcal Y$$

학습 알고리즘은 데이터를 넣으면 가설 하나를 내놓는다.

$$A(S)=h$$

(3) 데이터 생성 모델

입력 분포

$$\mathcal D:\ \text{distribution over }\mathcal X$$
학습자는 \(\mathcal D\)를 모른다. 어떤 분포든 가능하다고 놓고 시작한다.

정답 함수

$$f:\mathcal X\to\mathcal Y$$

이 장에서는 노이즈 없는 경우만 가정해

$$y_i=f(x_i),\qquad x_i\sim\mathcal D$$

(4) 성공의 척도: true error

새로 뽑힌 데이터에서 틀릴 확률이 진짜 성능이다.
$$L_{\mathcal D,f}(h):=\mathbb P_{x\sim\mathcal D}[h(x)\neq f(x)]$$
  • 미래 데이터에서 틀릴 확률
  • true error, generalization error, risk는 거의 같은 뜻으로 쓴다
학습자는 \(\mathcal D\)와 \(f\)를 모르고, 오직 표본 \(S\)만 본다. 그래서 이후의 모든 이야기는 “표본으로 미래 오차를 어떻게 추정할 것인가?”로 이어진다.
요소기호의미
입력 공간\(\mathcal X\)데이터가 존재하는 공간
출력 공간\(\mathcal Y\)라벨 공간
분포\(\mathcal D\)세상이 입력을 뽑는 방식
정답\(f\)진짜 규칙
데이터\(S\)관측한 표본
예측기\(h\)학습 결과
Core Idea

2.2 Empirical Risk Minimization

1) 진짜로 하고 싶은 것

$$\min_h\ L_{\mathcal D,f}(h)=\min_h\ \mathbb P_{x\sim\mathcal D}[h(x)\neq f(x)]$$

하지만 \(\mathcal D\)도 \(f\)도 모르므로 직접 풀 수 없다.

2) 대신 쓰는 것: training error

$$L_S(h):=\frac{|\{i\in[m]:h(x_i)\neq y_i\}|}{m}$$
  • 훈련 데이터에서 틀린 비율
  • empirical error, empirical risk, training error는 같은 뜻

3) ERM 규칙

“우리가 가진 정보는 훈련 데이터뿐이니, 거기서 제일 잘 맞는 모델을 고르자.”
$$\hat h\in\arg\min_h L_S(h)$$
Pitfall

2.2.1 Overfitting

반례 아이디어

  • 입력 공간은 회색 정사각형 전체
  • 분포 \(\mathcal D\)는 균등분포
  • 진짜 정답 함수 \(f\)는 파란 정사각형 내부만 1

암기형 예측기

$$h_S(x)=\begin{cases}y_i & \text{if }\exists i\in[m]\text{ s.t. }x=x_i\\0 & \text{otherwise}\end{cases}$$

훈련 샘플은 외우고, 나머지는 전부 0으로 예측한다.

훈련 오차

$$L_S(h_S)=0$$

훈련 데이터에는 완벽하다.

진짜 오차

$$L_{\mathcal D,f}(h_S)=\frac12$$

미래 데이터에는 형편없다.

훈련 오차를 낮췄다고 해서 true error까지 자동으로 작아지는 것은 아니다.
Fix

2.3 ERM with Inductive Bias

해결책은 ERM을 버리는 것이 아니라, 고를 수 있는 모델의 범위를 미리 제한하는 것이다.

Hypothesis class

$$\mathcal H\subset\{h:\mathcal X\to\mathcal Y\}$$
  • \(\mathcal H\)는 데이터 보기 전에 정해야 한다
  • 그래야 암기형 모델을 무한정 허용하지 않게 된다

제한된 ERM

$$\mathrm{ERM}_{\mathcal H}(S)\in\arg\min_{h\in\mathcal H}L_S(h)$$

왜 inductive bias인가

  • 어떤 형태의 함수만 허용한다는 사전 가정을 넣기 때문
  • 예: axis-aligned rectangle만 허용하면 “정답 영역이 네모일 것 같다”는 가정을 반영
핵심 tradeoff
  • \(\mathcal H\)를 좁히면 과적합 위험이 줄어든다
  • 하지만 너무 좁히면 진짜 \(f\)가 \(\mathcal H\) 안에 없을 수 있다
Guarantee

2.3.1 Finite Hypothesis Classes

나쁜 가설이 훈련에서 0오차처럼 보일 확률은 \((1-\varepsilon)^m\) 꼴로 지수적으로 줄어든다. 가설이 여러 개여도 union bound로 합칠 수 있다.
Step 1. ERM\(_\mathcal H\)
$$h_S\in\arg\min_{h\in\mathcal H}L_S(h)$$
Step 2. Realizability
$$\exists h^*\in\mathcal H\ \text{s.t.}\ L_{\mathcal D,f}(h^*)=0$$

그러면 \(L_S(h^*)=0\)이고 ERM이 고른 \(h_S\)도 훈련 오차 0을 가진다.

Step 3. 나쁜 가설 집합
$$\mathcal H_B=\{h\in\mathcal H:L_{\mathcal D,f}(h)>\varepsilon\}$$
Step 4. misleading samples
$$M=\{S:\exists h\in\mathcal H_B\ \text{s.t.}\ L_S(h)=0\}$$
$$\Pr[L_{\mathcal D,f}(h_S)>\varepsilon]\leq \Pr[M]$$
Step 5. Union bound
$$\Pr(M)\leq\sum_{h\in\mathcal H_B}\Pr(L_S(h)=0)$$
Step 6. i.i.d.로 확률 계산
$$\Pr(L_S(h)=0)=\big(1-L_{\mathcal D,f}(h)\big)^m\leq(1-\varepsilon)^m\leq e^{-\varepsilon m}$$
Step 7. 실패확률 상계
$$\Pr[L_{\mathcal D,f}(h_S)>\varepsilon]\leq |\mathcal H|e^{-\varepsilon m}$$
Step 8. 필요한 샘플 수
$$|\mathcal H|e^{-\varepsilon m}\leq\delta\ \Rightarrow\ m\geq\frac{\log(|\mathcal H|/\delta)}{\varepsilon}$$
  • \(|\mathcal H|\)가 클수록 데이터가 더 많이 필요
  • 더 작은 \(\varepsilon\), 더 작은 \(\delta\)를 원할수록 더 많은 샘플 필요
Wrap-up

최종 요약

  • 학습의 목표는 미래 오차 \(L_{\mathcal D,f}(h)\)를 줄이는 것이다.
  • 하지만 \(\mathcal D\)와 \(f\)를 모르므로, 훈련 오차 \(L_S(h)\)를 대신 최소화한다.
  • 제한 없는 ERM은 과적합을 일으킬 수 있다.
  • 그래서 hypothesis class \(\mathcal H\)를 미리 정하는 inductive bias가 필요하다.
  • \(|\mathcal H|<\infty\)이면 충분한 샘플 수에서 일반화가 보장된다.
